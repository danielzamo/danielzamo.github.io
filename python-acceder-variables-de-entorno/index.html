Python: variables de entorno

Acceder y modificar varibles de entorno, desde Python

Published Wen, Mar 2, 28283 by Daniel Oscar Zamo

Variables de entorno en Python

El mecanismo para establecer las variables de entorno depende de la plataforma. En Python se puede utilizar el módulo os. El mismo abstrae funcionalidad dependiente del sistema operativo, para poder trabajar con estas variables.

En tiempo de ejecución Python, mantiene todas las variables de entorno del programa en el diccionario os.environ como Objeto. Dicho objeto se llena cuando Python carga el módulo os. Si se intenta cambiar las variables de entorno después dara error.

Módulo so

Acceder a variables de entorno

>>>import os
>>>os.environ['PATH']
'/home/usuario/.linuxbrew/bin:/home/usuario/.linuxbrew/sbin:/home/usuario/.linuxbrew/bin:/home/usuario/.linuxbrew/sbin:/home/usuario/.local/bin:/home/usuario/bin:/usr/local/bin:/usr/bin:/usr/local/sbin:/usr/sbin'

Acceder mediante get

En caso de no existir la variable invocada, la anterior forma dara un error. Para que no ocurra lo anterior, se puede útilizar el método get, del siguiente modo:

>>> os.environ.get('VAR_NO_EXISTE')
>>> var=os.environ.get('VAR_NO_EXISTE')
>>> print(var)
None

Asignar un valor por defecto

Se puede utilizar la siguiente invocación con get, para asignar un valor por defecto si la variable no existe. Esto es:

>>> var=os.environ.get('VAR_NO_EXISTE','inicializada ahora')
>>> print(var)
inicializada ahora
>>>

Añadir/modificar variabres de entorno

En caso de necesitar añadir o modificar una variable de entorno, se utiliza el operador [] sobre os.environ.

>>> os.environ['PATH']
'/home/usuario/.linuxbrew/bin:/home/usuario/.linuxbrew/sbin:/home/usuario/.linuxbrew/bin:/home/usuario/.linuxbrew/sbin:/home/usuario/.local/bin:/home/usuario/bin:/usr/local/bin:/usr/bin:/usr/local/sbin:/usr/sbin'
>>> os.environ['PATH']='/bin/my-path'
>>> os.environ['PATH']
'/bin/my-path'

_Nota: _ Tenga en cuenta que los valores de las variables de entorno deben ser de tipo cadena. O sea, si se intenta asignar un valor que no sea del tipo cadena, el interprete dará un error en tiempo de ejecución.
